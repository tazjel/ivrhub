About

This is a skeleton Flask app using mongodb.

Roles

There are two roles: admins and normals. Admins can see things like the user directory. They can also edit anyone's profile (though not their password).

Signup flow

Users are asked to register and are sent the typical email with a confirmation link. There is another hurdle, however: after the confirmation is clicked, an admin is notified. The admin has to verify this person's identity and then toggle an entry in the webapp. Only then will full access to the site be granted. This is perhaps a bit silly and may be done away with in the future..

Source

Forgotten password flow

User visits /forgot and submits an email address. If that address is known, send it a message with a link to /forgot/<code>. This shows a two-input page asking for a new password. Save it and redirect to login.
